## 核心结论：企业微信自建应用是最佳方案

对"收文件 + 长任务 + 异步推结果"场景，**企业微信自建应用（HTTP 回调模式）是唯一在所有关键维度都满足的方案**。

## 企业微信关键技术点

### 必须避开的坑
⚠️ **用"自建应用 + 接收消息（HTTP 回调）"模式，不要用"智能机器人长连接"模式**——长连接模式收不了文件（GitHub openclaw#56140 确认）。

### 异步长任务标准架构
```
回调(5s超时) → 立即返回空串200 → 任务入队异步 → 完成后主动发消息API推送
```
- 发送应用消息接口(path/90236)**无48小时限制**，可任意时间主动推（vs 公众号的决定性优势）
- 频率：每用户每应用 ≤ 20条/分钟

### 文件收发
- 用户上传：回调含 media_id → 获取临时素材接口下载(path/90254)
- 结果下发：上传临时素材(path/90253)→media_id → 发文件消息(path/90236)
- 普通文件 ≤ 20MB，media_id 3天有效
- 回调必须幂等（5s重试3次，用MsgId去重）

### 注册
- 个人组建团队，身份证即可，免企业认证
- 需配置"企业可信IP"白名单
